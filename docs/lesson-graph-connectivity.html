8  Graph Connectivity

8.1 Connected and Disconnected Graphs

Look at Figure 7.1 again. Are there any disconnected pairs of nodes in the graph? The answer is no. If you pick any pair of nodes, they are either directly connected, or indirectly connected to one another via a path of some length. So how can we get two nodes to be disconnected in a simple undirected graph? The answer is that there has to be some kind of “gap” splitting the graph into two or more pieces, so that some set of nodes can no longer reach some of the other ones.

For instance, if you compare the undirected graphs Figure 8.1 (a) and Figure 8.1 (b) you can appreciate the difference between a connected graph and a disconnected graph. The difference between Figure 8.1 (a) and Figure 8.1 (b) is that the Figure 8.1 (b) graph is a subgraph of the Figure 8.1 (a) graph, in which the \(DF\) edge has been removed (an edge-deleted subgraph according to what we discussed in Chapter 4): \(G(b) = G(a) - DF\). Removing this edge has resulted in \(G(b)\) being disconnected. There is no way we can trace a path connecting any of the vertices in the \(\{F, G, H, I\}\) subset with any of the vertices in the \(\{A, B, C, D, E\}\) subset. This is different from Figure 8.1 (a) where we can trace a finite path between any two vertices, even if it is very long. Thus, we can come up with a formal definition of graph connectivity:

  • A graph is connected if there exists a (finite) path between each pair of nodes.

  • A graph is disconnected if there does not exist a path connecting every pair of nodes

8.2 Components and the Giant Component

When a graph goes from connected (like Figure 8.1 (a)) to disconnected (like Figure 8.1 (b)) it is split into a number of subgraphs that are themselves connected. For instance, in Figure 8.1 (b) the set nodes \(\{A, B, C, D, E\}\) form a connected subgraph of the larger disconnected graph. A connected subgraph that is part of a larger disconnected graph is called a component. In Figure 8.1 (b) there are two components, one formed by the connected subgraph with node set \(\{A, B, C, D, E\}\), and the other formed by the connected subgraph with node set \(\{F, G, H, I\}\).

It is possible to split a graph into multiple components not just two. For instance, take a look at Figure 8.2 (a). If we were to generate a subgraph from Figure 8.2 (a) by deleting the red and purple edges, we would end up with the Figure 8.2 (b); this graph is disconnected, and it features three separate components (connected subgraphs). If were to delete just the red edges from Figure 8.2 (a) and keep the purple edge, we would end up with the graph shown in Figure 8.2 (c). Like the Figure 8.2 (b), Figure 8.2 (c) is also disconnected, but it is split into two not three components.

Note that one of the connected components in Figure 8.2 (c) is way larger than the other one. The bigger connected component of graph Figure 8.2 (c) is of order ten, but the smaller component is only of order four. When a disconnected graph is split into multiple components, the component containing the largest number of nodes (the connected subgraph of the highest order) is called the giant component.

8.3 Bridges

We went from the connected graph shown in Figure 8.2 (a) to the disconnected graphs in Figure 8.2 (b) and Figure 8.2 (c) by removing or deleting particular edges (like the red and purple ones in Figure 8.2 (a)). These edges are clearly more important than the other blue ones, because they are responsible for keeping the whole graph together as a connected structure. Edges whose removal results in a graph going from being connected to disconnected are so important that they have a special name: bridges.

Thus, in a graph the bridges are (smallest in terms of cardinality) set of edges whose deletion results in the graph becoming disconnected. For instance, in Figure 8.1 (a) the \(DF\) edge is a bridge. In Figure 8.2 (a), on the other hand, any two of the threesome formed by the two red edges and the purple edge count as bridges. Deleting any two of these three edges disconnects the graph.

Note that in contrast to Figure 8.1 (a), Figure 8.2 (a) doesn’t have any one edge that is the bridge (the cardinality of the set of bridges is one). Instead, it takes deleting at least two edges from the red and purple set to disconnect it. For instance, if we were to remove just the purple edge, Figure 8.2 (a) would still be connected, and it would look like Figure 8.3 (a), which is a connected graph. The same goes for removing just one of the red edges. If were to do that, we would end up with the graph shown in Figure 8.3 (b), which is also a connected graph.

8.3.1 Local Bridges

In a classic paper on “The Strength of Weak Ties,” Mark Granovetter (1973) developed the concept of a local bridge. Recall from section Section 8.3, that a bridge is an edge (or set of edges) that if removed would completely disconnect the graph. There is another way of thinking of bridges in the context of shortest paths and this is with respect to what happens to the connectivity between particular pairs of nodes in the graph when a specific edge is removed. From this perspective, a bridge is an edge (or set of edges) that if removed would increase the length of the shortest paths between two sets of nodes from a particular number to infinity since, in a disconnected graph, the length of the path between nodes that cannot reach one another is indeed infinity! (\(\infty\)).

But of course, between a small number and infinity there’s a lot of in-between. That’s what the concept of a local bridge is intended to capture. A local bridge is an edge that if removed from the graph, would increase the length of the shortest path between a particular pair of nodes to a number that is less than infinity. This number is called the degree of the local bridge in question. Because a local bridge is always defined with respect to a particular pair of nodes, it is a triplet, involving two nodes and one edge.

For instance, in Figure 8.4 (a), with respect to nodes \(A\) and \(H\), the edge \(HK\) (pictured in purple) is a local bridge of degree 4. The reason for that is that, as shown in Figure 8.4 (b), when the edge \(HK\) is removed from the graph, the shortest path_ between nodes \(H\) and \(K\) increases from \(l_{HK} =1\) (\(H\) and \(K\) are adjacent in Figure 8.4 (a)) to \(l_{HK} = 4\), as given by the edge sequence \(\{GH, AG, AD, DK\}\) (pictured in red). Note that from the perspective of nodes \(D\) and \(H\), the purple \(HK\) edge is a local bridge of degree three, because \(l_{DH} = 2\) in Figure 8.4 (a) and \(l_{DH} = 3\) when the edge \(HK\) is removed from the graph in Figure 8.4 (b).

Granovetter, Mark S. 1973. “The Strength of Weak Ties.” American Journal of Sociology 78 (6): 1360–80.
